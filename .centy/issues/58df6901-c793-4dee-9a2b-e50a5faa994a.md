# Handle GitHub pull_request webhook event for auto-scoped runs

## Problem
Webhook only handles `push` events. No `pull_request` support → agents in CI can't auto-trigger a diff-scoped run when a PR is opened or updated.

## What to build
- Handle `pull_request` event type in `github_push` (or new handler)
- On PR open/synchronize: call `CreateRun(since_ref=base_sha, commit_sha=head_sha)`
- Store resulting run_id somewhere retrievable (e.g. return in webhook response, or post to PR)

## Files
- `server/src/webhook.rs`
- Proto: no changes needed — `CreateRun` already supports `since_ref`
