# Expose UpdateRun add_cases and metadata patch in UI

## Problem
`UpdateRun` supports `add_cases` (extend inline run scope mid-run) and patching `tester`/`environment`/`commit_sha` after creation. UI only exposes rename (`new_slug`).

## What to build
- 'Add cases to run' button on in-progress run — opens case picker, calls `UpdateRun(add_cases=[...])`
- Edit tester/environment/commit_sha inline on run detail view
- Response returns updated `pending` — avoid extra `GetPendingCases` call
