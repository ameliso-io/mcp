# 3rd-party import: ingest test results from external formats (e.g. Testmo CSV)

## Problem

No way to bring existing test data from external tools into Ameliso. Users migrating from Testmo (or similar) have historical runs/cases they can't import.

## Goal

Support importing test repositories/cases/runs from 3rd-party export formats.

## Example input

`/Users/ofek/Downloads/testmo-export-repository-5.csv` — Testmo repository export CSV.

## Scope

- Parse Testmo CSV export format (detect columns: suite, case name, steps, expected, tags, etc.)
- Map to Ameliso data model (repositories → suites → cases)
- CLI command: `ameliso import --from testmo --file <path>`
- UI: import button in repository settings or onboarding flow
- Validate/preview before commit (dry-run mode)
- Error reporting for unmapped/skipped rows

## Future formats

- JUnit XML
- Xray CSV
- Zephyr export
- Generic CSV with column mapping UI
