# First-run onboarding: guide from empty repo to first run

## Problem
Empty state shows 'No cases found.' with no next step. New users don't know: create a case → commit it → it syncs → create a run.

## What to build
- Detect zero-case state on first load
- Show onboarding checklist: (1) Create a case, (2) Commit to git, (3) Create a run
- Highlight completed steps as user progresses
- Dismissable once first run is completed

- `web/src/components/OverviewTab.tsx` or new `OnboardingBanner.tsx`
- `web/src/components/CasesTab.tsx` — empty state
