# Unit tests for mcp crate

Add unit tests for all MCP tool handlers in `mcp/src/main.rs`.

## What to test
- Each MCP tool handler: valid input → correct response
- Input validation and error messages
- Correct gRPC calls made to server for each tool
- Edge cases: missing required params, invalid repo

## Approach
- Spin up in-process gRPC server (same pattern as integration tests)
- Call MCP tool handler functions directly
- Assert on returned MCP `ToolResult` content

## Notes
- `mcp/src/main.rs` is ~41KB — largest file in project
- Map each exposed MCP tool to a test case
