# Append-only audit log for destructive actions

## Problem
No record of who deleted a case, finalized a run, or removed a repo. No compliance trail. Silent data loss with no forensics.

## What to build
- `audit_events` table: `(id, repo_id, actor_token_id, action, target_type, target_id, payload JSONB, created_at)`
- Write audit event on: `DeleteCase`, `BulkDeleteCases`, `DeleteRun`, `DeleteSuite`, `FinalizeRun`, `RemoveRepository`
- `GetAuditLog` RPC: paginated, filtered by `repo_id` and date range
- UI: audit log panel in settings page

## Files
- DB migration: `audit_events` table
- `server/src/service/` — write audit event in relevant handlers
- Proto + `web/`
