# Integrate agent_dylint_lints into pre-push lint validation

## Goal

Integrate https://github.com/tupe12334/agent_dylint_lints custom Dylint lints into the project and wire them into `make pre-push`.

## Lints available

- `assigning_clones`
- `copy_iterator`
- `max_lines_per_file`
- `max_lines_per_function`
- `no_expect`
- `no_unwrap`
- `tests_in_separate_files`
- `too_many_lines`
- `unused_async`
- `wildcard_imports`

## Implementation plan

### Step 1 — Wire up dylint

Add to workspace `Cargo.toml`:

```toml
[workspace.metadata.dylint]
libraries = [
    { git = "https://github.com/tupe12334/agent_dylint_lints" },
]
```

Install prerequisite:
```sh
cargo install dylint-link
cargo install cargo-dylint
```

### Step 2 — Initial suppression (start clean)

On first integration, add `#[allow(...)]` at crate/module level for ALL lints so CI passes immediately. This is intentional — suppresses everything via blanket directives so we can remove them one by one.

### Step 3 — Remove allow directives one by one

Tackle each lint individually in separate commits:
1. Remove the allow directive for the lint
2. Run `cargo dylint --all` to find violations
3. Fix all violations in code (no re-adding the allow)
4. Commit

Suggested order (least invasive first):
1. `wildcard_imports` — rename imports
2. `assigning_clones` — refactor clone assignments
3. `copy_iterator` — impl Iterator correctly
4. `unused_async` — remove unnecessary async
5. `no_unwrap` — replace `.unwrap()` with `?` or proper error handling
6. `no_expect` — replace `.expect()` with proper error handling
7. `too_many_lines` / `max_lines_per_function` — extract functions
8. `max_lines_per_file` — split files
9. `tests_in_separate_files` — move inline tests to `*_tests.rs` files

### Step 4 — Add `cargo dylint --all` to `make lint` and `make pre-push`

In `Makefile`, update `lint` target:
```makefile
lint:
    cargo clippy -- -D warnings
    cargo dylint --all
```

And ensure `pre-push` depends on `lint`.

## Definition of done

- `cargo dylint --all` passes with zero violations and zero `#[allow]` overrides for dylint lints
- `make pre-push` runs dylint as part of validation
- Each lint removal is a separate commit for reviewability
