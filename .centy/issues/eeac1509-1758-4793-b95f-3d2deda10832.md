# Add bulk case creation UI

## Problem
CasesTab only supports creating one case at a time. Proto has `BulkCreateCases` with optional `run_id` param to simultaneously add cases to an active run.

## Why it matters
Coverage gap discovery returns `uncovered_files` list — no efficient way to act on it. Agents need batch ops.

## What to build
- Multi-row case creation form or textarea (one title per line)
- Optional `run_id` selector to add created cases to an active run
- Wire to `BulkCreateCases` RPC
