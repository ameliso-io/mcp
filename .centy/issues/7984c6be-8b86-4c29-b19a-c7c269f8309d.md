# Add authentication middleware to gRPC endpoint

## Problem
gRPC server on port 50052 has zero authentication. Any client with network access can call any RPC — read or destroy any repo's data. No interceptor checks tokens or JWTs.

## What to build
- Tonic interceptor that validates a bearer token or API key on every RPC
- Token validation: verify against DB-stored hashed tokens (ties into API token management #37)
- Return `Status::unauthenticated` for missing/invalid credentials
- Exempt: `HandleGitHubCallback`, `GetGitHubInstallUrl` (public OAuth flow)

## Files
- `server/src/service/mod.rs` — add interceptor
- `server/src/main.rs` — wire interceptor into `AmelisoServiceServer`
