# Replace plain textarea with split-pane markdown editor for case body

## Why

Test case bodies are Markdown but the edit form uses a plain `<textarea>`. Users can't see formatting as they write, leading to raw asterisks in rendered output and poor adoption of structured steps.

## Implementation

Use existing libs already in the project:
- `marked` (already used in `MarkdownBody.tsx`) — render preview
- `DOMPurify` (already used in `MarkdownBody.tsx`) — sanitize rendered HTML
- No new dependencies

Split-pane: controlled `<textarea>` left + `<div dangerouslySetInnerHTML>` right, same pipeline as `MarkdownBody`.

## Done when

- Case create/edit form shows side-by-side editor: raw Markdown left, rendered preview right
- Preview uses `marked` + `DOMPurify` — identical output to `MarkdownBody.tsx` read view
- Toggle button collapses preview to full-width editor (mobile-friendly)
- `MarkdownBody` component reused directly for preview — no duplication of sanitize/render logic
- Zero new npm dependencies added
