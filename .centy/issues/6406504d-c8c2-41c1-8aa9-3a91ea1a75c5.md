# Quick record mode for active manual testing

## Problem
RunsTab during active testing is dense. Tester must navigate UI while also executing tests. No focused keyboard-first view.

## What to build
- 'Start testing' button enters quick-record mode: full case body + Pass/Fail/Block/Skip keyboard shortcuts (P/F/B/S)
- One case at a time, auto-advances on record
- Optional notes field (required only for Fail/Block, same as current rule)
- ESC exits back to full RunsTab

## Files
- `web/src/components/RunsTab.tsx` (or new `QuickRecordMode.tsx` sub-component)
