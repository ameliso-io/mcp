# Rerun button — clone previous run configuration

## Problem
No way to repeat a run. Tester must manually re-enter `tester`, `environment`, `suite`, `commit_sha`, and slug prefix every time.

## What to build
- 'Rerun' button on completed/aborted runs
- Pre-fills run creation form with previous run's `tester`, `environment`, `suite`/`cases`
- Updates `commit_sha` to current HEAD if git context available (see #41)
- Appends suffix to slug to avoid collision

- `web/src/components/RunsTab.tsx` — run list actions + creation form
