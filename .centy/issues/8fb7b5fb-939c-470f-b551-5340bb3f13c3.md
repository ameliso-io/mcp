# Mobile-responsive layout for field testing

## Problem
NavBar + tab layout breaks on small screens. Horizontal tab overflow, dense case lists, and small touch targets make the UI unusable on phones/tablets. Field testers recording results on mobile have no option.

## What to build
- Responsive NavBar: collapse tabs to hamburger menu on small screens
- CasesTab/RunsTab: stack card layout instead of table rows on mobile
- Larger tap targets on action buttons (min 44px)
- Test at 375px (iPhone SE) and 768px (iPad)

## Files
- `web/src/components/NavBar.module.css` + `NavBar.tsx`
- `web/src/components/RunsTab.module.css`, `CasesTab.module.css`
