# Add pagination to ListCases, ListRuns, ListSuites RPCs

## Problem
All `List*` RPCs return unbounded row sets. Large repos cause OOM on server and slow client renders. No cursor or page token exists.

## What to build
- Add `page_size` (int32) + `page_token` (string) to all three `List*` request messages
- Add `next_page_token` to response messages
- Server: keyset pagination on `(case_path/run_id/slug)` — no OFFSET
- Web: 'Load more' button or infinite scroll in CasesTab, RunsTab, SuitesTab

## Files
- Proto: `ListCasesRequest`, `ListRunsRequest`, `ListSuitesRequest` + responses
- `server/src/repo/` — paginated queries
- `web/src/components/CasesTab.tsx`, `RunsTab.tsx`, `SuitesTab.tsx`
