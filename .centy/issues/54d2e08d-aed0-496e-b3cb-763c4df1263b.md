# Add per-repo authorization model

## Problem
Even with auth (#75), all authenticated users can modify all repos. No access control boundary between repos or between read-only and read-write access.

## What to build
- Per-repo access table: `(repo_id, token_id, role)` where role = reader | writer | admin
- Interceptor or per-handler check: verify caller token has required role for target `repo_id`
- Admin role required for: `DeleteRun`, `BulkDeleteCases`, `RemoveRepository`
- Reader role sufficient for: all `Get*`, `List*` RPCs

## Files
- `server/src/service/mod.rs` — authorization helper
- DB migration: new `repo_permissions` table
