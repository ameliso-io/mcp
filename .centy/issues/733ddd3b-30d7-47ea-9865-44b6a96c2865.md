# Flakiness detection: flag cases that alternate pass/fail

## Problem
No visibility into unreliable test cases. Cases that alternate pass/fail across runs are wasting tester time and may indicate environment sensitivity.

## What to build
- Flakiness score per case: (alternating results / total runs) over last N runs
- Flag cases above threshold in CasesTab and coverage report
- Computable from existing `ListRuns` + `GetRun` data — no new RPCs
