# Split large files to ≤100 lines and enforce via ESLint incrementally

Reduce all `web/src` TypeScript/TSX files to ≤100 lines by splitting them one at a time, adding the `max-lines` ESLint rule after all files pass.

## Why

Files over 100 lines are hard to review and violate the project's single-responsibility intent. The ESLint config already has `max-lines: "off"` with a `TODO: enable and fix incrementally` comment — this issue tracks doing that.

## Process (per file)

1. Split file into focused sub-files (extract sub-components, helpers, hooks)
2. Run `pnpm lint` and `pnpm test` — both must pass
3. `git commit` with message like `refactor(web): split RunsTab into focused files`
4. `git push`
5. Move to next file

## Done when

- All non-generated `web/src` files are ≤100 lines
- `max-lines` and `max-lines-per-function` rules removed from the `"off"` list in `eslint.config.mjs` (or set to `["error", { max: 100 }]`)
- `pnpm lint` passes with the rule enabled
