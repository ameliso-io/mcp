# Enforce 100-line file limit — server (Rust)

## Problem

Files exceeding 100 lines signal SRP violations. Long Rust modules are harder to review, test, and reason about.

## Scope

- `server/**/*.rs`

Exclude:
- `*.pb.rs` (generated protobuf)
- Test fixtures
- `mod.rs` files that only re-export

## Implementation

Add CI step (or `just` recipe) that fails on any `.rs` file exceeding 100 lines:
```bash
find server/src -name "*.rs" ! -name "*.pb.rs" | while read f; do
  lines=$(grep -c '' "$f")
  if [ "$lines" -gt 100 ]; then
    echo "FAIL: $f ($lines lines)"
    exit 1
  fi
done
```

## Refactoring guidance

- Large modules → split into submodules by domain concept
- `impl` blocks with many methods → split trait implementations into separate files
- The limit is a signal, not a target

## Acceptance Criteria

- [ ] CI script blocks `.rs` files > 100 lines on every PR
- [ ] All existing violations refactored into smaller modules
- [ ] `just lint` or equivalent includes the check locally
- [ ] CLAUDE.md updated with the 100-line rule
