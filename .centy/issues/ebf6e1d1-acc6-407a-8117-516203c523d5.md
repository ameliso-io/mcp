# Add run comparison view to diff results between two runs

## Why

Testers need to spot regressions across releases. No way to see which cases changed status between run A and run B — requires manual cross-referencing.

## Done when

- "Compare" button on runs list lets user pick two finalized runs from same repo
- Comparison table shows cases with columns: case path | run A status | run B status | delta
- Delta column highlights: newly failing (red), newly passing (green), unchanged (gray)
- Comparison view is a URL-addressable route (e.g. `?compare=runA&runB=runB`) so it can be shared
- Uses existing `GetRun` / result data — no new RPC needed
