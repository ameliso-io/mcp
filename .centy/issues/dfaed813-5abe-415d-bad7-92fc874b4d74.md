# Add health/readiness endpoint to webhook server

## Problem
Webhook server on port 8080 has only `POST /webhook/github`. No `GET /health` or `GET /ready` route. Kubernetes/Docker liveness and readiness probes can't target it.

## What to build
- `GET /health` → 200 OK + `{"status":"ok"}`
- `GET /ready` → 200 if DB pool can acquire connection, 503 otherwise
- Add to existing Axum router in `main.rs`

## Files
- `server/src/main.rs` — extend `webhook_app` router
