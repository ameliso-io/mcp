# Enable gRPC reflection for developer tooling

## Problem
tonic-reflection not enabled. `grpcurl`, `grpc-client-cli`, and Postman gRPC mode don't work without local proto files. Blocks agent debugging and manual API exploration.

## What to build
- Add `tonic-reflection` crate
- Register all service descriptors in `main.rs`
- Verify `grpcurl -plaintext localhost:50052 list` works

- `server/Cargo.toml`
