# Allow FinalizeRun status override (force COMPLETED despite failures)

## Problem
UI always auto-detects finalized run status. Sometimes tester wants to force `COMPLETED` on a run with known/accepted failures. `FinalizeRunRequest.status` supports this but UI doesn't expose it.

## What to build
- Status selector on finalize dialog: 'Auto-detect' (default) / 'Force Completed' / 'Force Aborted'
- Pass selected `status` to `FinalizeRunRequest`

- `web/src/components/RunsTab.tsx` — finalize run dialog
