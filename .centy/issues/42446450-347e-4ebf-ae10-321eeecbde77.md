# Replace eprintln! with structured tracing (tracing crate)

## Problem
Server uses `eprintln!` throughout for logging. No log levels, no JSON output, no request IDs. Can't filter noise in production or correlate logs to requests.

## What to build
- Add `tracing` + `tracing-subscriber` crates
- Replace all `eprintln!` with `tracing::info!`, `tracing::warn!`, `tracing::error!`
- JSON formatter for production (`RUST_LOG` env var controls level)
- Add `tower-http` trace layer to Axum webhook router for request logging

## Files
- `server/Cargo.toml`
- `server/src/main.rs`
- All `server/src/` files with `eprintln!`
