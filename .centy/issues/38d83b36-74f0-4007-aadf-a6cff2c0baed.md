# Configure cargo-tarpaulin for Rust coverage reporting

Set up `cargo-tarpaulin` to measure and report Rust test coverage for all crates.

## Tasks
- Add `cargo-tarpaulin` to dev tooling (document install in README or Makefile)
- Add `make coverage` target that runs tarpaulin across the workspace
- Configure exclusions: generated proto code in `server/generated/`, `main.rs` binary entry points
- Target: ≥90% line coverage on `repo.rs`, `git.rs`, `github.rs`, `repos_store.rs`, `service.rs`

## Notes
- Tarpaulin needs `--workspace` flag to cover all crates
- Generated protobuf code should be excluded from coverage metrics
- Consider outputting lcov format for future CI integration
