# feat: save run-envs.toml to target repo

## Summary

Add `.ameliso/run-envs.toml` to target repos for predefined environment configuration. Replaces free-form environment text input with a structured combobox in the create-run form.

## TOML format

```toml
[[envs]]
name = "staging"
url = "https://staging.example.com"

[[envs]]
name = "production"
url = "https://production.example.com"
```

## Server changes

- **Cargo.toml**: add `toml = "0.8"` dependency
- **proto/types.proto**: add `RunEnv { name, url }` message
- **proto/service.proto**: add `GetRunEnvs` and `UpsertRunEnvs` RPCs + 4 messages
- **sync.rs**: add `RunEnv` struct, `run_envs_to_toml`, `parse_run_envs_toml`, `pull_run_envs`, `push_run_envs` (no DB — read/write via GitHub API directly)
- **service/mod.rs**: register new handlers; add `.toml` to `is_doc_file` so run-envs.toml changes don't appear as uncovered source files
- **service/get_run_envs.rs**: new handler (fetch from GitHub, return empty list if file absent)
- **service/upsert_run_envs.rs**: new handler (validate, fire-and-forget push via tokio::spawn)
- Tests for both handlers + sync roundtrip

## Web changes

- Replace environment `<input>` in create-run form with combobox: fetches `GetRunEnvs` on open, shows dropdown of configured envs, still allows custom free-form value
- Add `RunEnvsModal` component in Runs tab (gear icon near Create Run button): editable list of name+url rows, Save calls `UpsertRunEnvs`

## Patterns to follow

- `push_suite` fire-and-forget pattern (`sync.rs:280` + `create_suite.rs:44`)
- `upsert_file` for 409 SHA-conflict handling (`sync.rs:226`)
- `get_token_and_parts` for GitHub auth (`sync.rs:108`)
