# Replace 30s polling with SSE for live run updates

## Problem
Active run state and overview refresh via `useInterval` every 30s. Stale data between ticks; unnecessary requests when nothing changes.

## What to build
- Axum SSE endpoint `GET /events?repo_id=` streaming run state changes
- Server emits event on: result recorded, run finalized, run created
- Client subscribes via `EventSource`; replaces `useInterval` in RunsTab and OverviewTab
- Fall back to polling if SSE connection drops

## Files
- `server/src/main.rs` — add SSE route
- `web/src/hooks/useInterval.ts` — deprecate for run state
- `web/src/components/RunsTab.tsx`, `OverviewTab.tsx`
