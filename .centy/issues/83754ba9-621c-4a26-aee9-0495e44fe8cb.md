# Unit and integration tests for cli crate

Add tests for all CLI commands in `cli/src/main.rs`.

## What to test
- Argument parsing: each subcommand + flags parse correctly
- Command execution: each command produces correct output / side effects
- Error handling: missing args, bad repo path, server not running
- Output format: human-readable output matches expected format

## Approach
- Unit tests: test arg parsing with `clap`'s built-in test helpers
- Integration tests: spin up in-process server (same pattern as `server/tests/integration.rs`), run CLI commands against it, assert on stdout/stderr and gRPC state

## Notes
- `cli/src/main.rs` is ~27KB — many commands to cover
- Check `AMELISO_REPO` env var handling
