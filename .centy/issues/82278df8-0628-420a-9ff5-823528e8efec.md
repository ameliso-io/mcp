# Add request size limit to webhook endpoint

## Problem
`POST /webhook/github` has no body size limit. Oversized payload causes unbounded memory allocation — trivial DoS vector.

## What to build
- Apply Axum `DefaultBodyLimit::max(N)` to the webhook router (e.g. 1 MB)
- Return 413 for oversized payloads
- GitHub webhook payloads are well under 1 MB in practice

## Files
- `server/src/main.rs` — `webhook_app` router
