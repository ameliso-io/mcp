# Add coverage check to pre-push git hook

Enforce test coverage thresholds on every `git push` so coverage never regresses.

## Tasks
- Add `make coverage-check` target that runs `cargo-tarpaulin --fail-under <threshold>` across the workspace
- Add Vitest coverage check: `vitest run --coverage --reporter=json` with a minimum threshold in `vite.config.ts`
- Update `make pre-push` to run both coverage checks after the existing `build + test` steps
- The push must fail if either Rust or web coverage drops below the threshold

## Thresholds (starting points — tighten as coverage grows)
- Rust: 80% line coverage
- Web: 80% statement coverage

## Notes
- Current pre-push: `make pre-push` → `build + test` (husky hook in `.husky/pre-push`)
- `cargo-tarpaulin` must be installed (`cargo install cargo-tarpaulin`) — document in README
- Vitest coverage needs `@vitest/coverage-v8` or `@vitest/coverage-istanbul` as dev dep
- Depends on: cargo-tarpaulin setup issue (#12) and Vitest infra issue (#10)
