# Add real-time run progress via polling in RunsTab

## Why

Active runs show stale counts until user manually refreshes. Testers recording results in parallel see no updates. Breaks collaborative testing.

## Done when

- `RunsTab` polls `GetRun` / `GetPendingCases` every 3s when a run is `in_progress`
- Pass/fail/pending counts update live without full page reload
- Polling stops when run status becomes `finalized` or `aborted`
- Polling interval is a named constant (easy to tune)
- No polling on the runs list view (only active run detail)
