# Soft delete for cases, runs, and suites with restore

## Problem
`DeleteCase`, `DeleteRun`, `DeleteSuite` issue hard `DELETE` SQL. No recovery, no audit trail. One misclick destroys run history permanently.

## What to build
- Add `deleted_at TIMESTAMPTZ` to `cases`, `runs`, `suites` tables
- All `List*` / `Get*` queries filter `WHERE deleted_at IS NULL`
- New `RestoreCase`/`RestoreRun`/`RestoreSuite` RPCs (or `include_deleted` flag)
- UI: deleted items visible in archived view with restore button

## Files
- DB migrations
- `server/src/repo/` — all delete and list queries
- `web/src/components/` — archived view
