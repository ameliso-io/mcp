# Post run results as GitHub commit status check

## Problem
`commit_sha` is stored on runs but never written back to GitHub. After `FinalizeRun`, no signal reaches the PR or commit — CI has no pass/fail gate.

## What to build
- On `FinalizeRun`: if `run.commit_sha` is set, post a GitHub commit status via API (success/failure/pending)
- Status context: `ameliso/manual-tests`
- Description: summary counts (e.g. '12 passed, 1 failed')
- Link back to the run in Ameliso UI

## Files
- `server/src/service/finalize_run.rs`
- `server/src/github.rs` — add `post_commit_status` helper
