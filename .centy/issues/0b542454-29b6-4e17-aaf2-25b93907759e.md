# Auto-generate CLI and MCP code from proto via buf

## Problem

CLI (`cli/src/main.rs`) and MCP server (`mcp/src/main.rs`) are hand-written and manually kept in sync with `server/proto/ameliso/v1/service.proto`. Every new RPC requires touching three places: proto, CLI, and MCP.

## Goal

Use buf + gRPC to auto-generate the CLI command structure and MCP tool definitions directly from the proto service definition so the source of truth is `service.proto` alone.

## Tasks

- Configure `buf.gen.yaml` plugins for Rust code generation (tonic for gRPC client stubs)
- Evaluate buf plugins or custom protoc generators for CLI scaffolding (e.g. `protoc-gen-cobra` or equivalent Rust codegen)
- Evaluate buf plugins or custom generator for MCP tool definitions from proto RPCs
- Replace hand-written CLI subcommands in `cli/src/main.rs` with generated dispatch layer
- Replace hand-written MCP tool list in `mcp/src/main.rs` with generated tool definitions
- Wire generated gRPC client stubs into both CLI and MCP (replacing direct lib calls if needed)
- Add `buf generate` step to Makefile and pre-push hook

## Acceptance Criteria

- Adding a new RPC to `service.proto` + running `buf generate` produces the corresponding CLI subcommand and MCP tool with no manual edits to CLI or MCP source
- CI runs `buf generate` and fails if generated files are out of date
