# Split github.rs into per-function files with separate test files

Convert `server/src/github.rs` into a module folder `server/src/github/` where each logical unit lives in its own file, with tests in matching `*_test.rs` files.

## File layout

```
server/src/github/
  mod.rs                   — submodule declarations, pub re-exports, github_api_base(), ENV_LOCK
  config.rs                — GitHubConfig, config()
  config_test.rs           — config tests
  jwt.rs                   — JwtClaims, generate_jwt()
  jwt_test.rs              — jwt tests
  installation_token.rs    — InstallationTokenResponse, get_installation_token()
  installation_token_test.rs
  repos.rs                 — GitHubRepo, RepoListResponse, list_installation_repos(), get_repo()
  repos_test.rs
  installations.rs         — AppInstallation, list_app_installations()
  installations_test.rs
  compare.rs               — CompareResult + private types, compare()
  compare_test.rs
  files.rs                 — ContentResponse, get_file(), put_file(), delete_file()
  files_test.rs
```

## Key constraints

- `lib.rs` keeps `pub mod github;` unchanged — Rust resolves folder automatically via `mod.rs`
- `github_api_base()` stays private in `mod.rs`; submodules call it via `super::github_api_base()`
- `ENV_LOCK` declared `pub(crate)` in `mod.rs` under `#[cfg(test)]`; test files reference as `crate::github::ENV_LOCK`
- Each source file includes its test module: `#[cfg(test)] mod config_test;` etc.
- Delete `github.rs` after creating the folder

## Verification

Run `cargo test -p ameliso-server` — all existing tests must pass.
