# Enforce 100-line file limit — web (TypeScript/TSX)

## Problem

Files exceeding 100 lines signal SRP violations. Long components and hooks are harder to review, test, and reason about.

## Scope

- `web/src/**/*.{ts,tsx}`

Exclude:
- `*.generated.ts`
- Config files (`tailwind.config.ts`, `vite.config.ts`, etc.)
- `index.ts` barrel files (re-exports only, no logic)
- Test fixtures / snapshots

## Implementation

Add ESLint `max-lines` rule to `eslint.config.*`:
```js
"max-lines": ["error", { "max": 100, "skipBlankLines": true, "skipComments": true }]
```

Wire into CI so violations block merge. Optionally add to `.husky` / `lefthook` for local feedback.

## Refactoring guidance

- Large JSX trees → extract sub-components
- Hooks with many effects → split into focused hooks
- The limit is a signal, not a target — split by domain concept

## Acceptance Criteria

- [ ] ESLint `max-lines` rule enabled and erroring on violations
- [ ] All existing violations refactored
- [ ] CI enforces rule on every PR
- [ ] CLAUDE.md updated with the 100-line rule
