# Outbound webhook notification on run finalize

## Problem
No way to notify external systems (Slack, custom CI hook) when a run completes. GitHub status check (#58) covers GitHub only.

## What to build
- Per-repo configurable outbound webhook URL (stored in DB)
- On `FinalizeRun`: POST JSON summary to URL (run_id, status, pass/fail counts, commit_sha, tester)
- UI to configure URL in settings panel
- Retry on failure (3 attempts, exponential backoff)

## Files
- `server/src/service/finalize_run.rs`
- `server/src/repos_store.rs` — add `notification_url` field
- `web/` — settings UI for URL
