# Draft run mode — prepare scope before creating in DB

## Problem
Run is created in DB immediately on form submit. No way to assemble scope (suite, inline cases, metadata), review it, then commit. Accidental run creation requires delete + recreate.

## What to build
- 'Draft' state in run creation form: select scope and preview affected cases (via `GetAffectedCases` or `ListCases` filter) before submitting
- Show pending case count and list as preview
- Submit creates the run; Cancel discards draft (no DB write)
