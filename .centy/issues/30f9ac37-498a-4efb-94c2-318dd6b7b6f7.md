# Expose diff-scoped run creation in UI

## Problem
RunsTab form doesn't expose `use_last_run`, `since_ref`, or `changed_files` params even though the backend proto supports them.

## Why it matters
Core 'git compatible' value prop. Agents and testers can't create runs scoped to changed files without manual copy-paste.

## What to build
- Add `since_ref` text input (git ref or commit SHA)
- Add `use_last_run` toggle (auto-scope to cases affected since last run)
- Add `changed_files` textarea (paste raw git diff output)
- Wire to `CreateRun` RPC params
