# Migrate ESLint to eslint-config-agent (incremental)

## Goal

Migrate project ESLint setup to use `eslint-config-agent`. Each step = one small PR merged to main before next step starts.

---

## Steps

### Step 1 — Install dependencies
```
npm install --save-dev eslint eslint-config-agent
```
Push to main. Verify no broken CI.

### Step 2 — Add base `eslint.config.js` (or `.eslintrc`)
Create minimal config that extends `eslint-config-agent`. No rules changed yet, just wired up.
```js
// eslint.config.js
import agentConfig from 'eslint-config-agent';
export default [...agentConfig];
```
Push to main. Run `eslint .` locally to see violation count.

### Step 3 — Add ESLint script to `package.json`
```json
"lint": "eslint .",
"lint:fix": "eslint . --fix"
```
Push to main.

### Step 4 — Fix auto-fixable violations
Run `eslint . --fix`. Commit only auto-fixed changes (formatting, import order, etc.).
Push to main.

### Step 5 — Fix remaining violations (batch by file/component)
Work through remaining violations file by file. One commit per file or logical group.
Push to main per batch.

### Step 6 — Add ESLint to CI
Add `npm run lint` step to CI pipeline (GitHub Actions / Makefile).
Push to main.

### Step 7 — Enable stricter rules (optional)
After baseline passes: enable any additional rules from `eslint-config-agent` that were turned off.
Push to main.

---

## Notes
- Never fix and add features in same commit
- Each push must keep tests green
- If `eslint-config-agent` needs peer deps, install them in step 1
