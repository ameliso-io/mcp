# Add request correlation IDs for cross-RPC tracing

## Problem
No correlation IDs on requests. Can't trace a slow `GetCoverageReport` to its DB queries in production. Each `eprintln!` (or future log line) is isolated with no request context.

## What to build
- Tonic interceptor: extract or generate `x-request-id` header, attach to tracing span
- All DB queries within a request run under the same span
- Return `x-request-id` in gRPC response metadata for client-side correlation

## Files
- `server/src/main.rs` — interceptor
- `server/src/service/mod.rs`
