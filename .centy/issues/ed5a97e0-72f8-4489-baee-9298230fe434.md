# Import cases from CSV

## Goal

Allow users to bulk-import cases into Ameliso from a CSV file. Primary use case: migration from spreadsheets or other tools, and offline data collection workflows.

---

## User Flow

1. User opens import dialog (Cases list → "Import" button)
2. User uploads a `.csv` file (drag-and-drop or file picker)
3. System parses headers, auto-maps columns to case fields by name similarity
4. User reviews and adjusts column mapping via dropdowns
5. System runs validation pass, shows preview of first 10 rows with error highlights
6. User confirms — system imports valid rows, reports failures
7. User can download a failed-rows CSV to fix and re-import

---

## Acceptance Criteria

- [ ] Upload `.csv` file via file picker or drag-and-drop
- [ ] Auto-detect delimiter (comma, semicolon, tab) and encoding (UTF-8, UTF-8 BOM)
- [ ] Auto-map columns to case fields by header name; user can override mapping via UI
- [ ] Required fields (e.g. title) must be mapped before import can proceed
- [ ] Preview table shows first 10 rows with per-cell validation errors highlighted
- [ ] Summary: "X rows ready, Y rows have errors"
- [ ] Import options: **skip invalid rows** (import what's valid) vs **abort on any error**
- [ ] Progress indicator for files >100 rows; large imports run async
- [ ] Result screen shows: success count, failed rows with row number + reason
- [ ] Failed rows downloadable as a corrected-template CSV for re-import
- [ ] "Download CSV template" button exports headers + one example row
- [ ] Duplicate detection: configurable per import (skip / overwrite / error)

---

## Field Mapping Scope

Map to existing case fields:
- Title (required)
- Description
- Status
- Priority
- Tags
- Assignee (by name or email)
- Created date (optional override)
- Custom fields

---

## Technical Notes

**Server (Rust):**
- Use `csv` crate for streaming parse — do not load entire file into memory
- Validation layer returns per-row errors with row index and field name
- Batch insert; wrap in transaction for abort-on-error mode
- Background job (tokio task or queue) for large batches (>500 rows)
- API: `POST /api/cases/import` — multipart form, returns job ID for async polling
- `GET /api/cases/import/:job_id` — poll status + result

**Web (frontend):**
- Column mapping UI: table of [CSV column → case field dropdown]
- File size limit: warn at 5MB, hard reject at 20MB
- Highlight invalid cells in preview table
- Error report table: row #, column, value, reason
- Download error report as CSV

---

## Out of Scope (v1)

- Import from Google Sheets URL directly
- Excel (.xlsx) format
- Scheduled/recurring imports
- Import cases across multiple projects in one file
